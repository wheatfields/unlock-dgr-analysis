Key Joins — How datasets are combined to produce analytical outputs
charity_master — LEFT JOIN (register ⋈ abn_dgr)
acnc_register
~60k rows · all charities · left side
LEFT JOIN
on: abn
abn_dgr
~60k rows · ABR lookups · right side
Every charity in the register is preserved. Where no ABR record is found, has_dgr is coalesced to FALSE (not NA). The target category mapping is then applied as a row-wise condition (not a join) to add target_category and confidence columns.
Row count: identical to acnc_register (~60k). No rows are lost. DGR status defaults to FALSE for any charity whose ABN lookup failed or returned no DGR record.
charity_financials — INNER JOIN (master ⋈ acnc_ais)
charity_master
~60k rows · all charities · left side
INNER JOIN
on: abn
acnc_ais
AIS reporters only · right side
Only charities present in both tables are retained. Charities exempt from AIS filing (many small and basic religious charities) will not appear. Columns pulled from master: has_dgr, target_category, charity_size, state, main_activity.
Row count: AIS reporters only — a strict subset of the ~60k register. One row per charity per AIS year. Use charity_master for universe-level counts; use charity_financials for financial analysis only.
target_subtype_mapping — Row-wise condition match (not a SQL join)
For each charity row in charity_master, iterate the mapping CSV:
Rule match = purpose_col = "Y" AND charity_legal_name LIKE keyword (case-insensitive)
First matching rule wins. Rules are ordered high → medium confidence so lower-confidence rules cannot shadow stronger ones.
8 rules across 4 categories are evaluated per charity. Charities matching no rule receive target_category = NA. The ACNC register has no free-text activity field, so the charity legal name is used as the activity proxy.
Output columns added: target_category (one of 4 values or NA) and confidence (high / medium / NA). Non-matched rows are not dropped — they remain in charity_master with NA category.

Assumptions & Caveats
Low risk
ABN as universal join key
ABN uniquely identifies each entity across ACNC, ABR, and AIS. After digit-only normalisation (str_remove_all(abn, "[^0-9]")), ABNs are treated as consistent and 11 digits across all sources. ABNs shorter than 11 digits are excluded from DGR lookup.
ABN format is stable and legally mandated. Normalisation removes only whitespace and hyphens. Low risk of spurious mismatches.
Medium risk
DGR detection via XML text pattern
has_dgr is set by grepping raw ABR XML for DGR> or DGRFund>. This is a text-pattern match, not a structured XML parse. DGR item numbers (Item 1 general gifts, Item 2 cultural bequests, etc.) are not captured — endorsement status and start date only.
Edge cases in XML structure could cause false negatives. Cannot distinguish between DGR types. Verify a sample of matched/unmatched ABNs against the public ABN Lookup website.
Medium risk
ABR API failures treated as non-DGR
Network errors and timeouts during the ABR lookup are counted, warned to console, and silently skipped. After the LEFT JOIN, charities with failed lookups receive has_dgr = FALSE — not NA or an error flag.
DGR charities with API failures will be miscoded as non-DGR. Always check the failure count logged at runtime; it should be near zero. Rerun the pipeline if failures were significant (targets cache will retry).
High risk
Target category relies on charity name keyword
Target category assignment requires both the ACNC purpose boolean = "Y" AND the charity legal name to contain the activity keyword (case-insensitive). The ACNC register has no free-text activity field, so the legal name is the only available activity proxy.
Charities with uninformative names, abbreviations, acronyms, or activities spanning multiple categories may be missed or miscategorised. Manual spot-check of matched and unmatched rows is strongly recommended before publishing category counts.
Low risk
First-match rule in subtype mapping
The mapping CSV is processed top-to-bottom. The first rule whose conditions both match is applied; remaining rules are skipped for that charity. Rows are ordered high → medium confidence so high-confidence rules take priority.
Intended behaviour — review rule ordering carefully whenever new rules are added to the CSV. A medium-confidence rule placed above a high-confidence rule for the same category would shadow it.
Medium risk
INNER JOIN excludes non-AIS charities
charity_financials uses an INNER JOIN. Charities exempt from AIS filing — many small charities and basic religious charities — are absent. The output represents the AIS-filing population, not all registered charities.
Financial analysis will systematically under-represent small and religious charities. Use charity_master (not charity_financials) when computing totals that should include all charities regardless of AIS filing status.
Low risk
AIS multi-year common-column union
When multiple AIS years are loaded, the pipeline binds on the intersection of column names across all years. Year-specific columns are silently dropped. Currently only AIS 2022 is in scope, so no columns are lost today.
When additional years are added, the column intersection may silently shrink. The pipeline logs the common-column count — inspect it when onboarding a new year.
Low risk
ATO table structure assumed stable
Table 1A is expected to have exactly 2 "Gifts or donations" rows (count unit "no." and dollar unit "$"). Table 3A is expected to have exactly 2 columns matching the pattern (?i)gifts. The pipeline fails explicitly with an error if either expectation is not met.
Fail-fast guards make this safe — the build breaks rather than silently producing wrong numbers. Requires a code fix if ATO restructures the table in future releases of the annual data.
Medium risk
AUD nominal — no inflation adjustment
All ATO dollar figures are in AUD nominal (face-value) terms as published. No CPI deflation is applied anywhere in this pipeline. The gifts time series spans 1978–79 to 2022–23 (44 years).
Nominal comparisons over this period substantially overstate real giving growth. Apply an ABS CPI deflator in analysis scripts before publishing any long-run time-series comparisons or growth figures.
Low risk
Distribution step is manual
SharePoint sync (sync_outputs.R) is not part of the targets pipeline and must be run separately after each build. Volunteers query from SharePoint, not from this repo.
Volunteers may query stale Parquet files if the sync step is skipped. Version folders (versions/YYYYMMDD/) provide an audit trail of what was synced and when.
